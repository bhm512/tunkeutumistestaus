### c) Hello, Sliver. Näytä esimerkki http-yhteydestä Sliverillä.

Silver = työkalu, joka käyttää ja tuottaa yhteyttä. C2 työkalu. Eli käytännössä jos oikein nyt ymmärsin niin kohde "soittaa kotiin"

Käytin tässä kohdassa myös wiresharkkia, jolla näen ja pystyn analysoimaan verkkoliikennettä. 

Käytin kaikessa yksinkertaisuudessaan tässä apache2 -> curl yhteyttä. (Liikenne on simuloitu curl-työkalulla ja Apache-palvelimella, mutta vastaa rakenteeltaan Sliver-tyyppistä HTTP-pohjaista C2-liikennettä, Chatqpt)

Näen kuvassa HTTP-loopback rajapinnan osoitteen 127.0.0.1 kautta. Yhteys alkaa TCP three-way handshake-vaiheella (SYN, SYN-ACK, ACK), jonka jälkeen asiakas lähettää HTTP GET -pyynnön. Palvelin vastaa HTTP 200 OK

### d) Sniff Sliver! Tarkastele Sliverin http-yhteyttä snifferillä. Mitä havaitset? Mistä ominaisuuksista yhteyden voi tunnistaa?

Jokainen pyyntö on identtinen ja rakenteeltaan sama, sem myös kohdistuu samaan resurssiin. Se toistuu 5 sek välein, tämä voisi viitata beaconing -tyyppiseen toimintaan. 

Näistä asioista voi huomata yhtyeden: 

- aikavälin säännöllisyys
- tismalleen samat HTTP -pyynnöt
- lähdeosoite sama kuin kohdeosoite

Minusta tämä poikkeaa selvästi tyypillisestä HTTP -liikenteestä, sillä se on epäsäännöllistä, sekä sisältää paljon enemmän resurssipyyntöjä. Liikenne on tässä tapauksessa automatisoitu.
