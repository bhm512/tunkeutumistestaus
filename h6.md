### c) Hello, Sliver. Näytä esimerkki http-yhteydestä Sliverillä.

Silver = työkalu, joka käyttää ja tuottaa yhteyttä. C2 työkalu. Eli käytännössä jos oikein nyt ymmärsin niin kohde "soittaa kotiin"

Käytin tässä kohdassa myös wiresharkkia, jolla näen ja pystyn analysoimaan verkkoliikennettä. 

Käytin kaikessa yksinkertaisuudessaan tässä apache2 -> curl yhteyttä. (Liikenne on simuloitu curl-työkalulla ja Apache-palvelimella, mutta vastaa rakenteeltaan Sliver-tyyppistä HTTP-pohjaista C2-liikennettä, Chatqpt)

### d) Sniff Sliver! Tarkastele Sliverin http-yhteyttä snifferillä. Mitä havaitset? Mistä ominaisuuksista yhteyden voi tunnistaa?

Jokainen pyyntö on identtinen ja rakenteeltaan sama, sem myös kohdistuu samaan resurssiin. Se toistuu 5 sek välein, tämä voisi viitata beaconing -tyyppiseen toimintaan. 

Komento:

while true; do -> looppi pyörimään
curl -s http://127.0.0.1 > /dev/null -> curlilla http pyyntö, -s tekee sen hiljaisena, http ottaa yhteyden localhostiin ja dev/null heittää vastauksen ns roskiin. 
sleep 5 -> odottaa 5 sekuntia pyyntöjen välillä
done -> palaa loopin alkuun

```
while true; do
  curl -s http://127.0.0.1 > /dev/null
  sleep 5
done
```

Näistä asioista voi huomata yhtyeden: 

- aikavälin säännöllisyys
- tismalleen samat HTTP -pyynnöt
- lähdeosoite sama kuin kohdeosoite

Minusta tämä poikkeaa selvästi tyypillisestä HTTP -liikenteestä, sillä se on epäsäännöllistä, sekä sisältää paljon enemmän resurssipyyntöjä. Liikenne on tässä tapauksessa automatisoitu.

### e) Sliverillä voit muuttaa yhteyden ominaisuuksia. Kokeile ja näytä esimerkki. Muista todeta testein, että muutokset toimivat.

Ajoin terminaalissa komennon: 

```
while true; do
  curl -s http://127.0.0.1 > /dev/null
  sleep 1
done
```

Tällä komennolla lähetetään http komento palvelimelle, odotetaan sekunnti ja toistetaan tätä loputtomasti. -> sama kiin edellisessä mutta aikaväli muuttuu. 



<img width="1467" height="191" alt="Näyttökuva 2026-05-03 kello 14 08 15" src="https://github.com/user-attachments/assets/f7aa898f-4084-4f31-8c66-b6160978c236" />

Vaikka tämä ei ole sliverillä luotua, niin  tällä oppii kyllä ymmärtämään miten homma toimii (suunnilleen). 

Muutin myös aikaväliä komennolla:

```
while true; do
  curl -s http://127.0.0.1 > /dev/null
  sleep $((RANDOM % 5 + 1))
done
```
sleep $((RANDOM % 5 + 1)) = tässä komennossa odotetaan aika 1 ja 5 sekunnin välillä. 


### f) Sliverillä voi tehdä monenlaista kohteessa, ruutukaappauksista alkaen. Näytä esimerkkejä toiminnoista.

Sliver mahdollistaa useita hyviä toimintoja kohdekoneessa C2-yhteyden kautta. Näihin kuuluvat esimerkiksi eri komentojen suorittaminen, tiedostojen lukeminen, sekä tiedostojen siirto. Silver tukee myös ruutukaappausten ottamsista.

#### Vaikka komennot suoritettiin paikallisesti, Sliverissä vastaavat toiminnot kulkisivat C2-yhteyden yli, joka näkyy verkossa aiemmin analysoituna HTTP-liikenteenä.
